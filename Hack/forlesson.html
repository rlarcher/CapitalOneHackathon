Lesson: For Loops

Imagine you're teaching your little brother how to fix breakfast. A list of the steps would be something like:
1. Get two pieces of bread from the breadbox
2. Put the bread into the toaster
3. Toast bread
4. When toast pops out of the toaster, take it out and put it on the plate
5. Put butter on the toast and enjoy

Well, your little brother probably has seen your parents fix breakfast before, so he probably wouldn?t need all of these instructions listed out. But what if your brother had never seen anyone make toast? Each of those five steps has to be done once, so you can?t leave any of that information out.

What if you were making pancakes and the batter needed 10 teaspoons of flour? Would you include this information in your instructions like this:
1. Get the mixing bowl and bag of flour
2. Put a teaspoon of flour into the bowl
3. Put a teaspoon of flour into the bowl
4. Put a teaspoon of flour into the bowl
5. Put a teaspoon of flour into the bowl
6. Put a teaspoon of flour into the bowl
7. Put a teaspoon of flour into the bowl
8. Put a teaspoon of flour into the bowl
9. Put a teaspoon of flour into the bowl
10. Put a teaspoon of flour into the bowl
11. Put a teaspoon of flour into the bowl

Probably not. It would make a lot more sense to tell him:
1. Get the mixing bowl and a bag of flour
2. Put 10 teaspoons of flour in the bowl

When you work with a computer, you need give it the instructions for any process that you want to get done. Computer instructions are pretty similar to the instructions you?d give your little brother because you might have to write out separate, unique steps?like with the toast?or you can shorten a repetitive process by summarizing into one step, like with the flour.

If you were putting 10 teaspoons of flour into a bowl, you would need keep track of some pieces of important information like:
1. What you are doing (putting teaspoons of flour into a bowl)
2. How many teaspoons of flour have you put in so far? (0 when you start, but this number will increase as you put more spoonfuls in)
3. How many teaspoons of flour do you still have to put in? (10 when you start, but this number will decrease as you put more spoonfuls in)
?while actually putting each spoonful into the bowl.

When you start with an empty bowl, the process feels something like:
1. Number of teaspoons put in so far = 0. Number of teaspoons left = 10. Pour in a teaspoon of flour.
2. Number of teaspoons put in so far = 1. Number of teaspoons left = 9. Pour in a teaspoon of flour.
3. Number of teaspoons put in so far = 2. Number of teaspoons left = 8. Pour in a teaspoon of flour.
4. Number of teaspoons put in so far = 3. Number of teaspoons left = 7. Pour in a teaspoon of flour.
5. Number of teaspoons put in so far = 4. Number of teaspoons left = 6. Pour in a teaspoon of flour.
6. Number of teaspoons put in so far = 5. Number of teaspoons left = 5. Pour in a teaspoon of flour.
7. Number of teaspoons put in so far = 6. Number of teaspoons left = 4. Pour in a teaspoon of flour.
8. Number of teaspoons put in so far = 7. Number of teaspoons left = 3. Pour in a teaspoon of flour.
9. Number of teaspoons put in so far = 8. Number of teaspoons left = 2. Pour in a teaspoon of flour.
10. Number of teaspoons put in so far = 9. Number of teaspoons left = 1. Pour in a teaspoon of flour.
11. Number of teaspoons put in so far = 10. Number of teaspoons left = 0. Stop!

Computers use a pretty neat way to package all of this information into something called a for loop. A for loop is pretty useful in a situation like this, where you?re doing something a bunch of times and you want to keep track of how many times you did it. A for loop packages information together in a small, easy-to-follow way that looks strange at first, but is actually pretty simple and convenient.

The basic structure is:
for (a count of how many times you did an action and how many more times you need to do it)
{
what are you trying to do
}

In the context of putting teaspoons of flour in a bowl, it would be:
for(# of spoonfuls you?ve put in WHEN YOU START; # of total spoonfuls you want; how to adjust the number of spoonfuls you?ve put in so far as you add more flour)
{
add a spoonful of flour in
}

Some notes on the values in the previous skeleton:
1. Starting point: The number of spoonfuls you?ve put in when you start is usually 0. It wouldn?t really make sense to start the count at a number at, say, unless you?re writing this for loop to model a situation where your brother already put in 3 spoonfuls and you?re finishing up for him.
2. Ending point: The # of total spoonfuls that you need is pretty much what it seems. We need 10 total spoonfuls, so that?s the value we need.
3. How you got from start to finish: You currently have 3 teaspoons of flour in the bowl and then you add another. How did that number change? You just add 1 to 3. The same is true when you go from 5 to 6 teaspoons, 6 to 7, et cetera. You can symbolize this change by writing: teaspoons + 1.

But writing all of these words out takes a long time. What if we used the letter ?s? to symbolize the number of teaspoons we?ve put in? Now we can rewrite the information in the previous list as:
1. Starting point: int s = 0 (?int? just reminds us that you mean s is an integer, not a 0 symbol)
2. Ending point: s < 10 (this tells us that you keep going until you hit ten teaspoons, which is when you stop)
3. How you got from start to finish: s++. This means the same thing as s + 1, but it?s easier to type, especially if you?re writing this over and over again.

for (int s = 0; s < 10; s++){
put a spoonful of flour in the bowl;
